## Installation

```bash
npm install @noddde/core @noddde/engine
```

Or with yarn:

```bash
yarn add @noddde/core @noddde/engine
```

Or with pnpm:

```bash
pnpm add @noddde/core @noddde/engine
```

`@noddde/core` provides the type definitions, identity functions, and interfaces for defining your domain (aggregates, commands, events, queries, projections, sagas). `@noddde/engine` provides the runtime implementations (`configureDomain`, in-memory buses, persistence, and the event emitter event bus).

### ORM Adapters (optional)

For production persistence backed by a real database, install one of the ORM adapter packages:

```bash
# Drizzle ORM (SQLite, PostgreSQL, MySQL)
yarn add @noddde/drizzle drizzle-orm

# Prisma (any Prisma-supported database)
yarn add @noddde/prisma @prisma/client

# TypeORM (any TypeORM-supported database)
yarn add @noddde/typeorm typeorm reflect-metadata
```

Each adapter provides event-sourced persistence, state-stored persistence, saga persistence, and a database-backed unit of work — all wired through a single factory function. See [ORM Adapters](/docs/infrastructure/orm-adapters) for setup details.

## Requirements

- **Node.js** >= 18
- **TypeScript** >= 5.3

## TypeScript Configuration

noDDDe relies on TypeScript's strict mode for full type safety. Ensure your `tsconfig.json` has:

```json
{
  "compilerOptions": {
    "strict": true,
    "target": "ES2022",
    "module": "Node16",
    "moduleResolution": "Node16"
  }
}
```

Key settings:

- **`strict: true`** — Required for correct discriminated union narrowing and type inference
- **`target: ES2022`** or later — For modern JavaScript features used by the framework
- **`moduleResolution: "Node16"`** — For proper ESM/CJS resolution

## Verify Installation

Create a simple file to verify the installation:

```ts
import { DefineCommands, DefineEvents, defineAggregate } from "@noddde/core";

type MyCommand = DefineCommands<{ DoSomething: { value: string } }>;
type MyEvent = DefineEvents<{ SomethingDone: { value: string } }>;

type MyDef = {
  state: { value: string };
  events: MyEvent;
  commands: MyCommand;
  infrastructure: {};
};

const MyAggregate = defineAggregate<MyDef>({
  initialState: { value: "" },
  commands: {
    DoSomething: (command) => ({
      name: "SomethingDone",
      payload: { value: command.payload.value },
    }),
  },
  apply: {
    SomethingDone: (event) => ({ value: event.value }),
  },
});
```

If this compiles without errors, noDDDe is correctly installed and configured.
