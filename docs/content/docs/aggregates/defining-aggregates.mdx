## The defineAggregate Function

`defineAggregate<T>()` is the entry point for creating an aggregate. It accepts a single configuration object with three properties -- `initialState`, `commands`, and `apply` -- and returns the same object with full type inference across all boundaries.

```typescript
import { defineAggregate } from "@noddde/core";

export const MyAggregate = defineAggregate<MyAggregateDef>({
  initialState: { /* ... */ },
  commands: { /* ... */ },
  apply: { /* ... */ },
});
```

The generic parameter `T` is an `AggregateTypes` bundle that ties together the state, events, commands, and infrastructure types for your aggregate. This single type parameter is what makes the entire definition fully type-safe.

## Step-by-Step: Building an Aggregate

The recommended order for defining an aggregate is:

1. Define your **events** (what can happen)
2. Define your **commands** (what callers can request)
3. Define your **state** (what the aggregate tracks)
4. Bundle them into an **AggregateTypes** type
5. Call **defineAggregate**

### Step 1: Define Events

Events represent facts that have occurred. Use `DefineEvents` to declare them as a payload map:

```typescript
import { DefineEvents } from "@noddde/core";

type BankAccountEvent = DefineEvents<{
  BankAccountCreated: { id: string };
  TransactionAuthorized: {
    id: string;
    timestamp: Date;
    amount: number;
    merchant: string;
  };
  TransactionDeclined: {
    id: string;
    timestamp: Date;
    amount: number;
    merchant: string;
  };
  TransactionProcessed: {
    id: string;
    timestamp: Date;
    amount: number;
    merchant: string;
  };
}>;
```

Each key becomes the event's `name`, and the value becomes its `payload` type. The resulting type is a discriminated union of all event shapes.

### Step 2: Define Commands

Commands express intent. Use `DefineCommands` to declare them:

```typescript
import { DefineCommands } from "@noddde/core";

type BankAccountCommand = DefineCommands<{
  CreateBankAccount: void;
  AuthorizeTransaction: { amount: number; merchant: string };
}>;
```

Use `void` for commands that carry no payload beyond the target aggregate ID. Every command automatically includes a `name` and `targetAggregateId` field.

### Step 3: Define State

The state interface describes the data your aggregate tracks:

```typescript
interface BankAccountState {
  balance: number;
  availableBalance: number;
  transactions: Array<{
    id: string;
    timestamp: Date;
    amount: number;
    merchant: string;
    status: "pending" | "processed" | "declined";
  }>;
}
```

### Step 4: Create the AggregateTypes Bundle

Bundle everything into a single type that `defineAggregate` uses for inference:

```typescript
type BankAccountDef = {
  state: BankAccountState;
  events: BankAccountEvent;
  commands: BankAccountCommand;
  infrastructure: Infrastructure;
};
```

The `infrastructure` field defaults to the base `Infrastructure` type (an empty object). If your command handlers need access to services like loggers or clocks, specify a custom infrastructure type here (see [Using Infrastructure](#using-infrastructure) below).

### Step 5: Call defineAggregate

```typescript
import { defineAggregate } from "@noddde/core";

export const BankAccount = defineAggregate<BankAccountDef>({
  initialState: {
    balance: 0,
    availableBalance: 0,
    transactions: [],
  },

  commands: {
    CreateBankAccount: (command, _state, { logger }) => {
      logger.info(
        `Creating bank account ${command.targetAggregateId}`,
      );
      return {
        name: "BankAccountCreated",
        payload: { id: command.targetAggregateId },
      };
    },

    AuthorizeTransaction: (command, state, { logger }) => {
      const { amount, merchant } = command.payload;

      if (state.availableBalance < amount) {
        return {
          name: "TransactionDeclined",
          payload: {
            id: command.targetAggregateId,
            timestamp: new Date(),
            amount,
            merchant,
          },
        };
      }

      return {
        name: "TransactionAuthorized",
        payload: {
          id: command.targetAggregateId,
          timestamp: new Date(),
          amount,
          merchant,
        },
      };
    },
  },

  apply: {
    BankAccountCreated: (_event, _state) => ({
      balance: 0,
      availableBalance: 0,
      transactions: [],
    }),

    TransactionAuthorized: (event, state) => ({
      ...state,
      availableBalance: state.availableBalance - event.amount,
      transactions: [
        ...state.transactions,
        { ...event, status: "pending" as const },
      ],
    }),

    TransactionDeclined: (event, state) => ({
      ...state,
      transactions: [
        ...state.transactions,
        { ...event, status: "declined" as const },
      ],
    }),

    TransactionProcessed: (event, state) => ({
      ...state,
      balance: state.balance - event.amount,
      transactions: state.transactions.map((t) =>
        t.id === event.id
          ? { ...t, status: "processed" as const }
          : t,
      ),
    }),
  },
});
```

TypeScript enforces that:
- Every command name has a matching handler
- Every event name has a matching apply handler
- Command handlers return only events from the aggregate's event union
- Apply handlers return the exact state shape
- Infrastructure access is correctly typed

## Minimal Example

Not every aggregate needs infrastructure or complex state. Here is a minimal counter aggregate:

```typescript
import { DefineEvents, DefineCommands, defineAggregate } from "@noddde/core";

type CounterEvent = DefineEvents<{
  Incremented: { by: number };
  Decremented: { by: number };
}>;

type CounterCommand = DefineCommands<{
  Increment: { by: number };
  Decrement: { by: number };
}>;

type CounterDef = {
  state: { value: number };
  events: CounterEvent;
  commands: CounterCommand;
  infrastructure: {};
};

export const Counter = defineAggregate<CounterDef>({
  initialState: { value: 0 },

  commands: {
    Increment: (command) => ({
      name: "Incremented",
      payload: { by: command.payload.by },
    }),
    Decrement: (command, state) => {
      if (state.value - command.payload.by < 0) {
        // Return the event anyway -- business rule:
        // never go below zero, clamp to zero
        return {
          name: "Decremented",
          payload: { by: state.value },
        };
      }
      return {
        name: "Decremented",
        payload: { by: command.payload.by },
      };
    },
  },

  apply: {
    Incremented: (event, state) => ({
      value: state.value + event.by,
    }),
    Decremented: (event, state) => ({
      value: state.value - event.by,
    }),
  },
});
```

## Using Infrastructure

When command handlers need external services, define a custom infrastructure type and include it in your `AggregateTypes` bundle.

```typescript
import { Infrastructure } from "@noddde/core";

interface AuctionInfrastructure extends Infrastructure {
  clock: { now(): Date };
}

type AuctionEvent = DefineEvents<{
  AuctionCreated: { item: string; startingPrice: number };
  BidPlaced: { bidderId: string; amount: number; timestamp: Date };
  BidRejected: { bidderId: string; amount: number; reason: string };
  AuctionClosed: { winnerId: string | null };
}>;

type AuctionCommand = DefineCommands<{
  CreateAuction: { item: string; startingPrice: number; endsAt: Date };
  PlaceBid: { bidderId: string; amount: number };
  CloseAuction: void;
}>;

interface AuctionState {
  item: string;
  startingPrice: number;
  endsAt: Date | null;
  status: "pending" | "open" | "closed";
  highestBid: { bidderId: string; amount: number } | null;
}

type AuctionDef = {
  state: AuctionState;
  events: AuctionEvent;
  commands: AuctionCommand;
  infrastructure: AuctionInfrastructure;
};

export const Auction = defineAggregate<AuctionDef>({
  initialState: {
    item: "",
    startingPrice: 0,
    endsAt: null,
    status: "pending",
    highestBid: null,
  },

  commands: {
    CreateAuction: (command) => ({
      name: "AuctionCreated",
      payload: command.payload,
    }),

    PlaceBid: (command, state, { clock }) => {
      if (state.status !== "open") {
        return {
          name: "BidRejected",
          payload: {
            ...command.payload,
            reason: "Auction is not open",
          },
        };
      }

      if (state.endsAt && clock.now() > state.endsAt) {
        return {
          name: "BidRejected",
          payload: {
            ...command.payload,
            reason: "Auction has ended",
          },
        };
      }

      const minimum =
        state.highestBid?.amount ?? state.startingPrice;
      if (command.payload.amount <= minimum) {
        return {
          name: "BidRejected",
          payload: {
            ...command.payload,
            reason: `Bid must exceed ${minimum}`,
          },
        };
      }

      return {
        name: "BidPlaced",
        payload: {
          ...command.payload,
          timestamp: clock.now(),
        },
      };
    },

    CloseAuction: (_command, state) => ({
      name: "AuctionClosed",
      payload: { winnerId: state.highestBid?.bidderId ?? null },
    }),
  },

  apply: {
    AuctionCreated: (event) => ({
      item: event.item,
      startingPrice: event.startingPrice,
      endsAt: null,
      status: "open" as const,
      highestBid: null,
    }),

    BidPlaced: (event, state) => ({
      ...state,
      highestBid: {
        bidderId: event.bidderId,
        amount: event.amount,
      },
    }),

    BidRejected: (_event, state) => state,

    AuctionClosed: (_event, state) => ({
      ...state,
      status: "closed" as const,
    }),
  },
});
```

Notice how the `PlaceBid` handler destructures `{ clock }` from the third argument. The `AuctionInfrastructure` type ensures TypeScript knows exactly what services are available.

Also notice the `BidRejected` apply handler: it returns state unchanged. This is a valid pattern -- the event is still recorded (it is a fact that happened), but the state does not change as a result.

## Command Handlers Returning Multiple Events

A command handler can return an array of events when a single command triggers multiple state changes:

```typescript
commands: {
  CloseMonth: (command, state) => {
    const events: BankAccountEvent[] = [];

    // Process all pending transactions
    for (const txn of state.transactions) {
      if (txn.status === "pending") {
        events.push({
          name: "TransactionProcessed",
          payload: {
            id: txn.id,
            timestamp: new Date(),
            amount: txn.amount,
            merchant: txn.merchant,
          },
        });
      }
    }

    return events;
  },
}
```

## Async Command Handlers

Command handlers can be asynchronous when they need to call infrastructure services that return promises:

```typescript
commands: {
  AuthorizeTransaction: async (command, state, { fraudCheck }) => {
    const isSuspicious = await fraudCheck.evaluate(
      command.targetAggregateId,
      command.payload.amount,
    );

    if (isSuspicious) {
      return {
        name: "TransactionDeclined",
        payload: { /* ... */ },
      };
    }

    return {
      name: "TransactionAuthorized",
      payload: { /* ... */ },
    };
  },
}
```
