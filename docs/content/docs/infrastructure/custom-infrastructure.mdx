title: Custom Infrastructure
description: Defining, implementing, and providing custom infrastructure types for your domain.
---

Every domain has unique infrastructure needs. A banking system needs view repositories and loggers. An auction system needs a clock. A notification service needs an email client. noddde lets you define exactly what your domain needs as a TypeScript interface.

## Defining Your Infrastructure Type

Start by defining an interface that describes what your command handlers and projections need:

```ts
// Banking domain infrastructure
interface BankingInfrastructure {
  logger: Logger;
  bankAccountViewRepository: BankAccountViewRepository;
  transactionViewRepository: TransactionViewRepository;
}
```

```ts
// Auction domain infrastructure
interface AuctionInfrastructure {
  clock: Clock;
}
```

Keep it minimal. Only include what handlers actually use. If only one handler needs a specific dependency, it still belongs in infrastructure — the alternative (importing directly) breaks testability.

## Implementing Interfaces

Each member of your infrastructure type should be an interface with concrete implementations:

```ts
// Interface
interface Logger {
  info(message: string): void;
  error(message: string): void;
  warn(message: string): void;
}

// Production implementation
class ConsoleLogger implements Logger {
  info(message: string) { console.log(message); }
  error(message: string) { console.error(message); }
  warn(message: string) { console.warn(message); }
}

// Test implementation
class SilentLogger implements Logger {
  info(_message: string) {}
  error(_message: string) {}
  warn(_message: string) {}
}
```

```ts
// Interface
interface Clock {
  now(): Date;
}

// Production implementation
class SystemClock implements Clock {
  now(): Date { return new Date(); }
}

// Test implementation
class FixedClock implements Clock {
  constructor(private readonly time: Date) {}
  now(): Date { return this.time; }
}
```

This pattern — interface plus swappable implementations — is what makes noddde handlers testable. See the [Clock Pattern](/docs/patterns/clock-pattern) for a detailed walkthrough.

## Providing Infrastructure

Infrastructure is provided via the `provideInfrastructure` factory in the [domain configuration](/docs/domain-configuration/infrastructure):

```ts
const domain = await configureDomain<BankingInfrastructure>({
  // ...
  infrastructure: {
    provideInfrastructure: () => ({
      logger: new ConsoleLogger(),
      bankAccountViewRepository: new InMemoryBankAccountViewRepository(),
      transactionViewRepository: new InMemoryTransactionViewRepository(),
    }),
  },
});
```

The generic parameter `<BankingInfrastructure>` on `configureDomain` ensures the factory return type matches your infrastructure definition. TypeScript will report an error if you forget a member or return the wrong type.

## Using Infrastructure in Handlers

### In Command Handlers

Destructure what you need from the third parameter:

```ts
commands: {
  CreateBankAccount: (command, _state, { logger }) => {
    logger.info(`Creating bank account ${command.targetAggregateId}`);
    return {
      name: "BankAccountCreated",
      payload: { id: command.targetAggregateId },
    };
  },
},
```

### In Projection Event Handlers

Destructure from the second parameter:

```ts
eventHandlers: {
  BankAccountCreated: (event, { bankAccountViewRepository }) => {
    bankAccountViewRepository.insert({
      id: event.id,
      balance: 0,
    });
  },
},
```

### In Query Handlers

Also the second parameter:

```ts
const getBankAccountById: QueryHandler<BankingInfrastructure, GetBankAccountByIdQuery> =
  async (query, { bankAccountViewRepository }) => {
    return bankAccountViewRepository.getById(query.id);
  };
```

## Infrastructure in the Type System

Your infrastructure type flows through `AggregateTypes`:

```ts
type BankAccountDef = {
  state: BankAccountState;
  events: BankAccountEvent;
  commands: BankAccountCommand;
  infrastructure: BankingInfrastructure; // ← here
};

export const BankAccount = defineAggregate<BankAccountDef>({
  // TypeScript now knows the infrastructure parameter
  // type in all command handlers
});
```

If you destructure `{ logger }` in a command handler and your infrastructure type does not have a `logger` property, TypeScript will flag it at compile time.
