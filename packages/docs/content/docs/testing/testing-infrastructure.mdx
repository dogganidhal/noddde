Infrastructure injection is what makes noddde handlers testable. This page covers patterns for creating test-friendly infrastructure implementations.

## The Clock Pattern

The most common testing challenge in event-sourced systems is non-deterministic behavior — typically caused by `new Date()` in command handlers. The [Clock Pattern](/docs/patterns/clock-pattern) solves this.

### The Problem

```ts
// This handler produces different results on every invocation
commands: {
  PlaceBid: (command, state, infrastructure) => {
    const now = new Date(); // Non-deterministic!
    if (now > state.endsAt) {
      return { name: "BidRejected", payload: { reason: "Auction has ended" } };
    }
    // ...
  },
},
```

### The Solution

Define a `Clock` interface and inject it:

```ts
interface Clock {
  now(): Date;
}

class SystemClock implements Clock {
  now(): Date { return new Date(); }
}

class FixedClock implements Clock {
  constructor(private readonly time: Date) {}
  now(): Date { return this.time; }
}
```

Use `FixedClock` in tests:

```ts
describe("PlaceBid", () => {
  const auctionEndsAt = new Date("2024-06-01T00:00:00Z");

  it("rejects bids after auction ends", () => {
    const pastEndTime = new FixedClock(new Date("2024-06-02T00:00:00Z"));

    const result = Auction.commands.PlaceBid(
      {
        name: "PlaceBid",
        targetAggregateId: "auction-1",
        payload: { bidderId: "alice", amount: 600 },
      },
      { ...auctionState, endsAt: auctionEndsAt },
      { clock: pastEndTime },
    );

    expect(result).toMatchObject({
      name: "BidRejected",
      payload: { reason: "Auction has ended" },
    });
  });

  it("accepts bids before auction ends", () => {
    const beforeEndTime = new FixedClock(new Date("2024-05-15T00:00:00Z"));

    const result = Auction.commands.PlaceBid(
      {
        name: "PlaceBid",
        targetAggregateId: "auction-1",
        payload: { bidderId: "alice", amount: 600 },
      },
      { ...auctionState, endsAt: auctionEndsAt, highestBid: null, startingPrice: 500 },
      { clock: beforeEndTime },
    );

    expect(result).toMatchObject({ name: "BidPlaced" });
  });
});
```

## Mock Infrastructure Objects

For infrastructure with multiple members, create a reusable mock factory:

```ts
function createMockBankingInfrastructure() {
  return {
    logger: {
      info: vi.fn(),
      error: vi.fn(),
      warn: vi.fn(),
    },
    bankAccountViewRepository: {
      getById: vi.fn(),
      insert: vi.fn(),
      update: vi.fn(),
    },
    transactionViewRepository: {
      listByBankAccountId: vi.fn(),
    },
  };
}

describe("BankAccount commands", () => {
  const infra = createMockBankingInfrastructure();

  it("logs on CreateBankAccount", () => {
    BankAccount.commands.CreateBankAccount(
      { name: "CreateBankAccount", targetAggregateId: "acc-1" },
      BankAccount.initialState,
      infra,
    );

    expect(infra.logger.info).toHaveBeenCalledWith(
      "Creating bank account acc-1",
    );
  });
});
```

## Generalizing the Pattern

The Clock pattern generalizes to any non-deterministic dependency:

| Dependency | Interface | Production | Test |
|---|---|---|---|
| Time | `Clock { now(): Date }` | `SystemClock` | `FixedClock` |
| Random IDs | `IdGenerator { generate(): string }` | `UUIDGenerator` | `SequentialIdGenerator` |
| External APIs | `PaymentGateway { charge(...) }` | `StripeGateway` | `MockPaymentGateway` |
| File system | `FileStorage { read(...), write(...) }` | `DiskStorage` | `InMemoryStorage` |

The pattern is always the same:
1. Define an interface for the capability
2. Create a production implementation
3. Create a test implementation with deterministic behavior
4. Add it to your infrastructure type

## Integration Testing

For slice tests with multiple components, use `testDomain` from `@noddde/testing`:

```ts
import { testDomain } from "@noddde/testing";

describe("Banking domain integration", () => {
  it("creates a bank account and authorizes a transaction", async () => {
    const { domain, spy } = await testDomain({
      aggregates: { BankAccount },
      projections: { BankAccount: BankAccountProjection },
      infrastructure: createMockBankingInfrastructure(),
    });

    await domain.dispatchCommand({
      name: "CreateBankAccount",
      targetAggregateId: "acc-1",
    });

    await domain.dispatchCommand({
      name: "AuthorizeTransaction",
      targetAggregateId: "acc-1",
      payload: { amount: 100, merchant: "Store" },
    });

    expect(spy.publishedEvents).toHaveLength(2);
  });
});
```

For full control over persistence and bus implementations, use `configureDomain` from `@noddde/engine` directly. See [Testing Domains](/docs/testing/testing-domains) for when to choose each approach.
