title: Testing Projections
description: Testing strategies for both legacy and functional projection styles.

Projections build read models from event streams. Testing them ensures your views are correctly constructed from the events your aggregates produce.

## Testing ProjectionV2 (Functional)

`ProjectionV2` reducers are pure functions — the easiest thing to test:

```ts
import { describe, it, expect } from "vitest";
import { BankAccountV2 } from "./projection";

describe("BankAccountV2 projection", () => {
  const emptyView = { id: "", balance: 0, transactions: [] };

  it("initializes on BankAccountCreated", () => {
    const result = BankAccountV2.reducer(emptyView, {
      name: "BankAccountCreated",
      payload: { id: "acc-123" },
    });

    expect(result).toEqual({
      id: "acc-123",
      balance: 0,
      transactions: [],
    });
  });

  it("updates balance on TransactionProcessed", () => {
    const view = { id: "acc-123", balance: 0, transactions: [] };

    const result = BankAccountV2.reducer(view, {
      name: "TransactionProcessed",
      payload: {
        id: "txn-1",
        timestamp: new Date("2024-01-15"),
        amount: 100,
        merchant: "Store",
      },
    });

    expect(result.balance).toBe(100);
    expect(result.transactions).toHaveLength(1);
  });

  it("ignores TransactionDeclined", () => {
    const view = { id: "acc-123", balance: 500, transactions: [] };

    const result = BankAccountV2.reducer(view, {
      name: "TransactionDeclined",
      payload: {
        id: "txn-2",
        timestamp: new Date(),
        amount: 1000,
        merchant: "Expensive",
      },
    });

    expect(result).toBe(view); // Same reference, unchanged
  });
});
```

### Testing Full Event Stream Replay

You can test a complete event sequence by folding:

```ts
it("builds a complete view from event stream", () => {
  const events = [
    { name: "BankAccountCreated" as const, payload: { id: "acc-1" } },
    { name: "TransactionProcessed" as const, payload: { id: "t1", timestamp: new Date(), amount: 500, merchant: "Salary" } },
    { name: "TransactionProcessed" as const, payload: { id: "t2", timestamp: new Date(), amount: -50, merchant: "Coffee" } },
  ];

  const view = events.reduce(BankAccountV2.reducer, emptyView);

  expect(view.id).toBe("acc-1");
  expect(view.balance).toBe(450);
  expect(view.transactions).toHaveLength(2);
});
```

## Testing Legacy Projections

Legacy projections require mock infrastructure since event handlers receive it as a parameter:

```ts
import { describe, it, expect, vi } from "vitest";
import { BankAccountProjection } from "./projection";

describe("BankAccountProjection", () => {
  const mockRepository = {
    insert: vi.fn(),
    update: vi.fn(),
    getById: vi.fn(),
  };

  const mockInfrastructure = {
    logger: { info: () => {}, error: () => {}, warn: () => {} },
    bankAccountViewRepository: mockRepository,
    transactionViewRepository: { listByBankAccountId: vi.fn() },
  };

  it("inserts a new view on BankAccountCreated", () => {
    BankAccountProjection.eventHandlers.BankAccountCreated?.(
      { id: "acc-123" },
      mockInfrastructure,
    );

    expect(mockRepository.insert).toHaveBeenCalledWith({
      id: "acc-123",
      balance: 0,
    });
  });
});
```

## Testing Query Handlers

Query handlers also receive infrastructure:

```ts
describe("getBankAccountByIdQueryHandler", () => {
  it("returns the account from repository", async () => {
    const mockRepo = {
      getById: vi.fn().mockResolvedValue({ id: "acc-1", balance: 500 }),
    };

    const result = await getBankAccountByIdQueryHandler(
      { id: "acc-1" },
      { bankAccountViewRepository: mockRepo } as any,
    );

    expect(result).toEqual({ id: "acc-1", balance: 500 });
    expect(mockRepo.getById).toHaveBeenCalledWith("acc-1");
  });
});
```

## Choosing What to Test

| Projection Style | What to Test | Mocks Needed? |
|---|---|---|
| `ProjectionV2` reducer | Event → View transformations | No |
| `Projection` event handlers | Side effects (repository calls) | Yes |
| `Projection` query handlers | Return values from infrastructure | Yes |

For `ProjectionV2`, you are testing pure transformations. For legacy `Projection`, you are testing interactions with infrastructure.
